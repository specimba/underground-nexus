Quick Reference Commands

  • Reset Portainer to set up initial login and password (if Portainer has timed out on first set-up):
    • sudo docker restart Olympiad0
  • Update Underground Nexus (The first command will take you to the root terminal of the entire Underground Nexus called Athena0, which is a red Kali Linux powered CLI – it is also possible to access Athena0 from Portainer to run the “sh” / shell script. Only run this script from Athena0.):
    • sudo docker exec -it Athena0 bash
    • sh /nexus-bucket/underground-nexus-update.sh
  • Access Kali Linux Athena0 shell:
    • sudo docker exec -it Athena0 bash
  • Reset Pi-hole password:
    • sudo docker exec Inner-DNS-Control sudo pihole -a -p
  • Access Kubectl for Kubernexus:
    • sudo docker exec -it k3d-KuberNexus-server-0 sh
    • kubectl cluster-info
  • Test Kubernetes by deploying Portainer
    • kubectl apply -n portainer -f https://downloads.portainer.io/ce2-16/portainer-lb.yaml
    • kubectl get services --all-namespaces
    • kubectl get pods --all-namespaces
  • URL to test access to the KuberNexus Portainer
  • To enter a pod’s shell in Kubernetes from inside server-0, use kubectl.
    • kubectl exec -it kubectl exec -it -- /bin/sh
  • NOTE: To use helm with kubernetes, the Athena0 shell may be used to configure kubectl to talk to KuberNexus using k3d to configure kubectl
  • NOTE: The root Underground Nexus container’s shell that contains the deploy-olympiad.sh script also has kubectl and helm fully configured for super admins to use
